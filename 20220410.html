    # 手把手帶你使用 VuePress 搭建線上技術文件

    # VuePress 簡介

    • VuePress 是一種靜態網站生成器，底層由Vue、VueRouter、Webpack 驅動。
    • 在構建時，會創建一個服務端渲染（SSR）的版本，然後通過虛擬訪問每一條路徑來渲染對應的HTML。（類似Nuxt.js）
    • 使用 Markdown 語法撰寫頁面及內文。
    • 支援在 Markdown 檔中使用 Vue 的模板語法和組件。
    • 支援多語系
    • 支援 RWD
    • ...

    # 快速上手

    1. 創建新目錄夾且進入
    mkdir my-vuepress && cd my-vuepress
    1. 初始化
    yarn init # npm init
    1. 安裝VuePress
    yarn add -D vuepress # npm install -D vuepress
    1. 建立第一篇文章
    mkdir docs && echo '# Hello TED-101讀書會好棒棒' > docs/index.md
    1. 添加 Build VuePress 的指令在 package.json
    {
  "scripts": {
    "docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs"
  }
}
    1. 啟動
    yarn docs:dev # npm run docs:dev

    VuePress 會在 http://localhost:8080 (opens new window) 啟動一個熱重載的開發服務器。

    # 目錄結構

    VuePress 遵循 “約定優於配置” 的原則，推薦的目錄結構如下：

    意思就是，底層的東西，大部份原本需要人工設定的東西，VuePress 都搞定了，我們不用去花時間設定路由、樣式、編譯 Markdown 什麼的，VuePress 統包。But，如果你有更多的需求要滿足，那也可以透過自己寫的配置（config）去達成

    • docs/.vuepress: 用于存放全局的配置、组件、静态资源等。
    • docs/.vuepress/components: 该目录中的 Vue 组件将会被自动注册为全局组件。
    • docs/.vuepress/theme: 用于存放本地主题。
    • docs/.vuepress/styles: 用于存放样式相关的文件。
    • docs/.vuepress/styles/index.styl: 将会被自动应用的全局样式文件，会生成在最终的 CSS 文件结尾，具有比默认样式更高的优先级。
    • docs/.vuepress/styles/palette.styl: 用于重写默认颜色常量，或者设置新的 stylus 颜色常量。
    • docs/.vuepress/public: 静态资源目录。
    • docs/.vuepress/templates: 存储 HTML 模板文件。
    • docs/.vuepress/templates/dev.html: 用于开发环境的 HTML 模板文件。
    • docs/.vuepress/templates/ssr.html: 构建时基于 Vue SSR 的 HTML 模板文件。
    • docs/.vuepress/config.js: 配置文件的入口文件，也可以是 YML 或 toml。
    • docs/.vuepress/enhanceApp.js: 客户端应用的增强。

    # 基本配置

    .
├─ docs
│  ├─ index.md
│  └─ .vuepress
│     └─ config.js
└─ package.json

    在 docs 目錄下建立 .vuepress/config.js，內容如下：

    module.exports = {
  title: 'TED-101 讀書會',
  description: '讀書會好棒棒',
	base: '/ted101_share/',              // 預設值是 /
	dest: 'docs',                        // 預設值是 .vuepress/dist
	head: [
		['link', { rel: 'icon', href: '/img/favo.ico' }]
	],
}

    # 主題配置

    # theme

    • 类型: string
    • 默认值: undefined

    当你使用自定义主题的时候，需要指定它。

    # themeConfig

    • 類型: Object
    • 預設值: {}

    為目前的主題提供配置。

    # 設置首頁

    默認的主題提供了一個首頁（Homepage）的佈局 (用於 這個網站的主頁)。想要使用它，需要在根目錄下的 README.md (opens new window) 的 YAML front matter 指定 home: true。以下是一個如何使用的例子：

    • YAML front matter

      Frontmatter | VuePress (opens new window)

    ---
home: true
heroImage: /hero.png
heroText: Hero 標題
tagline: Hero 副標題
actionText: 快速上手 →
actionLink: /guide/
features:
- title: Vuepress 好棒棒
  details: 以 Markdown 為中心的項目結構
- title: Vue 驅動
  details: Vue + webpack 就是好棒棒
- title: 高性能
  details: 總而言之 好棒棒
footer: MIT Licensed | Copyright © 2018-present Evan You
---

    # 導航欄

    • 導航欄鏈結
    module.exports = {
  themeConfig: {
    nav: [
      { text: '首頁', link: '/' },
      { text: '關於', link: '/about' },
      { text: '外部鏈結', link: 'https://google.com' },
    ]
  }
}
    • 導航欄Logo
    module.exports = {
  themeConfig: {
    logo: '/img/logo.png',
  }
}

    # 側邊欄

    • 側邊欄分組
    module.exports = {
  themeConfig: {
    sidebar: [
      {
        title: '指南',   // 必要
        path: '/foo/',      // 選填，標題地餓跳轉鏈結
        collapsable: false, // 選填，預設是 true
        sidebarDepth: 1,    // 選填，預設是 1
        children: [
          '/guide/overview',
					'/guide/quickstart'
        ]
      },
      {
        title: '分組 2',
        children: [ /* ... */ ]
      }
    ]
  }
}

    # 頁面滾動

    module.exports = {
  themeConfig: {
    smoothScroll: true
  }
}

    # 主題安裝

    https://vuepress-theme-reco.recoluan.com/ (opens new window)

    # 安裝

    npm install vuepress-theme-reco --save-dev

# or

yarn add vuepress-theme-reco

    # 引用

    // .vuepress/config.js

module.exports = {
  theme: 'reco'
}

    • 主委加碼

      # 手把手帶你使用VuePress搭建個人部落格

    # 配置修改為部落格類型

    // .vuepress/config.js

module.exports = {
  themeConfig: {
    type: 'blog'
  }
}

    # 設置作者

    // .vuepress/config.js
module.exports = {
  themeConfig: {
    author: '丹尼斯',          // 作者名稱
		authorAvatar: '/liu.png'。 // 作者頭像
  }
}

    # 添加文章發表時間軸

    module.exports = {
  themeConfig: {
    nav: [
      { text: 'TimeLine', link: '/timeline/', icon: 'reco-date' }
    ]
  }
}

    新增此配置後，在撰寫文章時，必須添加日期在 frontmatter 才會有效果。
